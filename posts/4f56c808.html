Hexo部署到Coding和Gitee

发表于2021-04-11|更新于2021-04-30|教程
|字数总计:961|阅读时长:2分钟|评论数:

前言

去年部署到Coding上是看的Yafine的教程，如今已过时效，网上也搜不到合适的教程，不得已只能自己摸索，踩了些坑总算是成了。

部署到Coding

注册登录

Coding官网

现在Coding被收入腾讯云名下，你所注册的账号是可以用来登录腾讯云的。

所以如果已有腾讯云账号且实名了的情况下，为了方便统一管理，建议注册时用微信扫码登录，这样你的Coding账户就和已有的腾讯云账号绑定了，之后部署时不需要再实名认证了

添加SSH公钥

然后打开C:\Users\你的电脑账户名\.ssh,用文本编辑器打开 id_rsa.pub，将里面的内容复制粘贴到对应位置

然后打开 cmd 控制台输入 ssh -T git@e.coding.net，第一次建立连接会要求信任主机，输入相关的用户名或者密码，在控制台输入验证，验证成功即可。

创建项目

在左侧菜单栏中选择项目，再点击页面右上角创建项目

然后选择DevOps项目

第一次创建时会自动创建一个引导项目，返回后再创建一个即可

创建代码仓库

进入项目后，在左侧菜单栏中选择代码仓库，再点击页面右上角创建代码仓库

之前代码仓库名称时需要和用户名一致的，保险起见可以将仓库名设为用户名

配置博客根目录配置文件

进入代码仓库中复制下仓库地址

然后再执行 hexo clean && hexo g -d 部署到 coding

静态网站构建

打开你的 coding 项目管理页,选择网站托管->新建网站

选择静态网站

按图示操作

网站类型选择静态网站，不要选Hexo，没有备案域名的情况下节点只能选香港

确定后会自动进行部署，当出现如下的页面效果，就说明部署成功。

进入后访问所给的地址就能进入网站了

自定义域名

进入如下页面添加域名

然后按照他所提示的，添加后到域名托管商更改解析记录

我的域名是阿里的，配置如下

DNS解析缓存需要一定时间，请耐心等待几分钟

接着点击证书状态，跳转到腾讯云SSL证书控制台，点击域名验证

接着按提示到域名解析商添加解析记录，耐心等待后邮箱会收到证书颁发的邮件

申请证书时要把github的解析暂停，不然很可能证书颁发不了

证书颁发成功后Coding会自动重新部署静态网站，时间可能较长，等待两三分钟后就能通过域名访问网站了，若长时间无反应可手动部署

查看部署状态方法

部署到Gitee

  • 码云官网进行注册，注册完成后，创建仓库

  • 添加秘钥

同coding

  • 复制项目地址，配置本地文件

然后执行 hexo clean && hexo g -d，部署成功后，点击服务 —>Gitee Pages

选择启动即可部署

小结

  • 主要是找教程时受到误导，以为部署到Coding上时只能用腾讯云的解析服务，还得把阿里的DNS改成腾讯的，在腾讯DNS 解析 DNSPod平台折腾来折腾去，尝试几次后发现可以直接用域名商的解析，不需要那么复杂。

  • gitee访问速度很快，自定义域名需要付费，所以我主要就部署在Coding和GitHub上，并使用双线部署，国内Coding,国外GitHub。
